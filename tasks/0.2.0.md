# TUR GUI 0.2.0 Plan

## Purpose

Turn the `0.1.0` GUI MVP into a credible day-to-day desktop downloader.

`0.1.0` proves that:

- `tur-rs` can power the GUI directly
- the Tauri shell and event bridge work
- basic download control is viable

`0.2.0` should make the app feel reliable and intentional rather than merely functional.

---

## Release Goal

Ship a stronger desktop release for Linux/macOS/Windows that:

- survives app restarts more gracefully
- handles larger task lists cleanly
- improves task control and status clarity
- exposes the most important downloader actions users expect
- hardens cross-platform behavior and packaging truth

This is still not the “ultimate GUI release.” It is the first serious usability-and-trust release.

---

## Non-Goals for 0.2.0

- browser challenge solving
- embedded browser/webview auth extraction
- system tray daemon mode
- deep theming system or plugin marketplace
- advanced scheduler/debug knobs for normal users
- mobile targets

---

## Phase 1 — Runtime Validation Closure

### Goal

Close the remaining gap from `0.1.0`: real GUI runtime validation, not just builds.

### Work

- perform and document:
  - one real download through the GUI
  - one pause/resume cycle
  - one cancel cycle
  - one session-aware path if available
- record any bugs exposed by those flows and fix them
- update task docs/changelog notes accordingly

### Success criteria

- `0.1.0` style flows are proven in actual GUI usage

---

## Phase 2 — Task Persistence and Restore

### Goal

Make the app more resilient across restarts.

### Work

- persist frontend-side task metadata needed for a sane restart experience
- on startup:
  - restore visible task rows from prior session
  - clearly distinguish restored finished tasks from live active tasks
- if `tur-rs` resumable state is available for stopped tasks, surface it cleanly in the GUI
- prevent duplicate/confusing row creation after restart

### Success criteria

- restarting the app does not make the task list feel disposable or incoherent

---

## Phase 3 — Better Task Management

### Goal

Make the download list usable once there are many rows.

### Work

- add:
  - clear completed
  - clear failed
  - remove row
  - retry failed
- add a collapsible per-task detail panel for richer runtime visibility
- improve filtering:
  - all
  - active
  - paused
  - completed
  - failed
- add simple sort controls where helpful:
  - newest
  - status
  - progress

If `tur-rs` exposes enough runtime information, the expanded task detail should show:

- per-connection/worker rows
- worker status such as:
  - connecting
  - downloading
  - stalled
  - retrying
  - finished
- per-connection speed / progress / assigned range where available
- task-level derived values such as:
  - remaining bytes
  - ETA
  - effective active connection count

### Success criteria

- the task list remains manageable after repeated downloads and failures
- expanded task details provide useful download-manager-style diagnostics without cluttering the compact view

---

## Phase 4 — File and Folder Actions

### Goal

Add the obvious actions users expect after a download completes.

### Work

- per-row actions for:
  - open containing folder
  - reveal in file manager where supported
  - open file where safe/applicable
- add graceful fallback when a target file no longer exists
- keep these actions platform-aware and conservative

### Success criteria

- completed downloads are actionable from the GUI without manual filesystem hunting

---

## Phase 5 — Session UX Hardening

### Goal

Make session-aware downloading clearer and less error-prone.

### Work

- improve the advanced/session section:
  - clearer help text
  - show when session fields are active for a task
- validate custom headers better
- validate cookie-file input better
- make session-related failures more visible and understandable in task rows

### Success criteria

- users can tell when a task is using session context and why it failed if that context is wrong

---

## Phase 6 — Settings and Defaults

### Goal

Reduce repetitive form entry for normal usage.

### Work

- add minimal persisted settings:
  - default download directory
  - default autostart preference
  - optional default connection/profile choices if later exposed
- keep settings small and product-focused
- avoid building a huge preferences surface

### Success criteria

- the app remembers the important defaults without becoming settings-heavy

---

## Phase 7 — Cross-Platform Packaging Truth

### Goal

Make the desktop product feel publishable across the major OS targets.

### Work

- verify platform build behavior where practical:
  - Linux local run
  - macOS/Windows compile/package checks if available in CI or collaborator environments
- verify generated icons and bundle metadata
- check autostart behavior by platform and document caveats
- tighten Tauri config / permissions if anything is broader than necessary

### Success criteria

- the app is not just “Tauri code that builds on one machine”; it is treated as a cross-platform desktop product

---

## Phase 8 — UI Polish Without Bloat

### Goal

Make the MVP shell feel deliberate and calmer to use.

### Work

- improve hierarchy and spacing in the task board
- tighten empty states and failure states
- make progress visuals more legible
- reduce noisy visual repetition
- keep performance acceptable on long lists

### Success criteria

- the app feels like a downloader product, not a dressed-up form demo

---

## Suggested Order

1. Runtime validation closure
2. Task persistence and restore
3. Better task management
4. File and folder actions
5. Session UX hardening
6. Settings and defaults
7. Cross-platform packaging truth
8. UI polish without bloat

---

## Final Assessment

`0.2.0` should be the release that makes `tur` GUI feel trustworthy for repeated everyday use.

If done correctly, it gives you:

- a stronger desktop downloader identity
- better persistence and task hygiene
- clearer session-aware behavior
- a much better base before bigger features like browser import, tray mode, or deeper integrations
